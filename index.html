Curso Completo: Aprende Node.js y Express - Curso desde Cero

Resumen de una línea: Este curso enseña los fundamentos de Node.js y Express para el desarrollo web backend, desde la instalación y conceptos básicos hasta la creación de un servidor y API funcionales.

Puntos clave:

  • Introducción a Node.js: Entorno de ejecución de JavaScript, eventos síncronos y asíncronos.
  • Manejo de módulos en Node.js: Importación y exportación, módulos built-in.
  • Desarrollo de servidores con Node.js y Express: Creación de rutas, manejo de solicitudes HTTP (GET, POST, PUT, DELETE), parámetros URL y query.
  • Uso de npm: Instalación y desinstalación de paquetes, manejo de dependencias.
  • Trabajo con JSON: Conversión entre objetos JavaScript y formato JSON, uso en solicitudes y respuestas HTTP.

Resumen del curso:

Este curso proporciona una introducción completa al desarrollo backend con Node.js y Express. Comienza con una explicación detallada de Node.js, incluyendo su naturaleza como entorno de ejecución de JavaScript orientado a eventos asíncronos, la diferencia entre eventos síncronos y asíncronos, y la importancia de la programación asíncrona para la creación de aplicaciones escalables y en tiempo real.

Se explora el concepto de módulos, su importancia para la organización y reutilización del código, y cómo importar y exportar elementos entre módulos. Se cubren los módulos built-in de Node.js como http, fs, process, events, y timers, mostrando ejemplos prácticos de su uso.

El curso continúa con una introducción a npm (Node Package Manager), explicando conceptos como paquetes, módulos y dependencias. Se guía al estudiante en la creación de un archivo package.json, la instalación y desinstalación de paquetes, y el manejo de dependencias de desarrollo.

Se dedica una sección significativa a JSON (JavaScript Object Notation), su sintaxis, y su uso en la transmisión de datos entre el cliente y el servidor. Se muestran ejemplos de cómo convertir objetos JavaScript a JSON y viceversa utilizando las funciones JSON.stringify() y JSON.parse().

La parte central del curso se centra en el desarrollo de servidores con Node.js y, posteriormente, con Express. Se explica el modelo cliente-servidor y el protocolo HTTP, incluyendo los métodos HTTP (GET, POST, PUT, DELETE) y los códigos de estado HTTP. Se construye un servidor básico con el módulo http de Node.js para comprender los fundamentos, y luego se introduce Express para simplificar el proceso de creación de rutas y manejo de solicitudes.

Se enseña cómo definir rutas utilizando diferentes métodos HTTP y cómo extraer información de la URL, incluyendo parámetros de ruta y parámetros query. Se profundiza en el manejo de solicitudes POST para agregar nuevos recursos, PUT para actualizar recursos, PATCH para actualizar propiedades específicas, y DELETE para eliminar recursos. Se utilizan ejemplos prácticos para ilustrar cada concepto, incluyendo la creación de una API simple para gestionar información de cursos.

Finalmente, se introduce Nodemon para automatizar el reinicio del servidor al detectar cambios en los archivos, mejorando la eficiencia del desarrollo. El curso concluye con una revisión de los conceptos clave y un desafío para el estudiante de expandir el proyecto creado.

NodeJS - Desarrollo Web

¿Qué es NodeJS?

Node.js es un entorno de ejecución de JavaScript que permite usar este lenguaje fuera del navegador, especialmente para crear servidores y aplicaciones backend. Está basado en el motor V8 de Google Chrome y utiliza un modelo orientado a eventos y programación asíncrona, lo que facilita manejar muchas operaciones concurrentes sin bloquear el sistema.

Diferencias entre JavaScript en frontend y en backend

JavaScript en el frontend se ejecuta dentro del navegador del usuario y se usa para manejar la interacción con la interfaz gráfica, validar formularios, animaciones, etc. En el backend (con Node.js), JavaScript corre en el servidor para procesar datos, manejar bases de datos, autenticar usuarios y responder a solicitudes HTTP.

Ventajas y desventajas de usar JavaScript en backend

Ventajas: Es un lenguaje conocido para muchos desarrolladores, permite usar el mismo lenguaje en frontend y backend, facilita la creación de aplicaciones rápidas y escalables con su modelo asíncrono.

Desventajas: No es ideal para tareas que requieren mucho procesamiento intensivo (CPU), su modelo asíncrono puede ser difícil de entender para principiantes y la gestión de callbacks/promesas puede complicar el código.

Instalación y configuración de Node.js

  1. Ir al sitio oficial: nodejs.org
  2. Elegir la versión LTS (recomendada para la mayoría).
  3. Descargar el instalador y seguir los pasos.
  4. Una vez instalado, abrir una terminal y verificar:
    • node -v: muestra la versión de Node.js
    • npm -v: muestra la versión del gestor de paquetes npm

Paso a paso: Importar y exportar módulos en Node.js

  1. Crear un archivo para exportar:

    Por ejemplo, exportar.js con una función y una variable:

    // exportar.js
function saludar(nombre) {
    return `Hola, ${nombre}!`;
}
let despedida = "¡Hasta luego!";
module.exports = { saludar, despedida };
  2. Crear un archivo para importar:

    Por ejemplo, importar.js donde importamos lo anterior:

    // importar.js
let { saludar, despedida } = require('./saludos');
console.log(saludar('Juan'));
console.log(despedida);
  3. Ejecutar el archivo principal:

    En la terminal, ejecuta node importar.js y observa el resultado:

¿Qué es npm y quién lo mantiene?

npm (Node Package Manager) es el gestor de paquetes oficial para Node.js. Permite instalar, actualizar y gestionar librerías o módulos que facilitan el desarrollo. npm es mantenido por la empresa npm, Inc., que ahora forma parte de GitHub/Microsoft.

¿Qué es Yarn y cómo se diferencia de npm?

Yarn es otro gestor de paquetes para Node.js, desarrollado por Facebook para mejorar algunos aspectos de npm como la velocidad, la seguridad y la gestión de dependencias. Yarn usa un archivo yarn.lock que garantiza versiones fijas, y tiene comandos y funcionamiento similar a npm.

Diferencias entre yarn.lock y package-lock.json

  • yarn.lock: Archivo que fija las versiones exactas de las dependencias instaladas con Yarn.
  • package-lock.json: Archivo equivalente pero usado por npm para garantizar reproducibilidad de instalaciones.

Comandos npm útiles

  • npm start: Ejecuta el script "start" definido en package.json para iniciar la aplicación.
  • npm test: Ejecuta los tests definidos en package.json.
  • npm uninstall <paquete>: Desinstala un paquete.
  • npm update: Actualiza los paquetes instalados a su última versión compatible.
  • npm list: Muestra los paquetes instalados en el proyecto.

¿Para qué sirven las banderas -flags- en npm?

  • -g: Instala un paquete de forma global en el sistema (disponible en cualquier proyecto o terminal).
  • --save: (Históricamente) Guardaba el paquete en las dependencias de package.json, ahora npm lo hace por defecto.
  • -D o --save-dev: Guarda el paquete en las dependencias de desarrollo (devDependencies), para usar solo en desarrollo.

Módulos npm y Uso Práctico

¿Para qué sirven estos módulos npm?

  • express: Framework para construir servidores y APIs web en Node.js de forma rápida y sencilla.
  • mongodb: Driver oficial para conectar Node.js con bases de datos MongoDB.
  • mongoose: ORM para MongoDB que facilita definir esquemas y trabajar con datos.
  • path: Módulo built-in que ayuda a manejar rutas y directorios en el sistema de archivos.
  • router: Middleware para definir rutas en aplicaciones Express (aunque a veces viene integrado en Express).
  • nodemon: Herramienta para reiniciar automáticamente el servidor cuando detecta cambios en los archivos.
  • cors: Middleware que habilita Cross-Origin Resource Sharing para controlar acceso a recursos desde otros dominios.
  • http: Módulo built-in para crear servidores HTTP sin usar frameworks externos.
  • multer: Middleware para manejar subida de archivos (multipart/form-data) en Express.
  • morgan: Middleware para registrar peticiones HTTP en la consola, útil para depuración.
  • prettier: Formateador de código para mantener un estilo consistente.
  • debug: Utilidad para generar mensajes de depuración de forma sencilla y controlada.
  • jade (pug): Motor de plantillas para generar HTML dinámico desde plantillas en Node.js.
  • passport: Middleware para autenticación, compatible con muchas estrategias (Google, Facebook, local, etc.).

¿Existen paquetes para validar formularios y exportar o importar archivos?

Sí, existen muchos paquetes que facilitan estas tareas:

  • Validación de formularios: express-validator, joi.
  • Exportar a PDF: pdfkit, puppeteer (generar PDFs a partir de HTML).
  • Exportar a Excel: exceljs, xlsx.
  • Cargar desde PDF o Excel: pdf-parse para PDFs, xlsx para Excel.

3 módulos interesantes para probar

  • dotenv: Permite cargar variables de entorno desde un archivo .env para separar configuración sensible del código.
  • socket.io: Facilita la comunicación en tiempo real (WebSocket) entre cliente y servidor.
  • bcrypt: Librería para encriptar contraseñas y manejar seguridad en autenticación.

¿Cómo se utiliza un módulo instalado?

Una vez instalado con npm install nombre-modulo, lo importamos en nuestro archivo JavaScript con:

let modulo = require('nombre-modulo');

Luego usamos las funciones o clases que provee ese módulo en nuestro código.

¿Cómo indicar que un módulo es solo para desarrollo?

Al instalar un módulo que solo se usará durante el desarrollo (por ejemplo, nodemon o prettier), se usa la bandera -D o --save-dev, así:

npm install nombre-modulo -D

Esto agrega el módulo en la sección devDependencies de package.json, y no se instala en producción con un comando como npm install --production.
